# Control Assessment

This artifact is part of a GRC portfolio project based on a realistic SaaS company running in AWS and handling customer data.

I created this control assessment to practice evaluating whether key security and governance controls actually exist, how well they are working, and where improvement is still needed. Instead of assuming a control is effective just because it exists on paper, I focused on whether there was enough evidence to support it.

The scenario reflects common cloud realities: AWS IAM, logging and alerting, user access changes, vendor dependencies, and backup recovery expectations.

Each row represents:
- what control is being reviewed
- which risk the control is meant to address
- what the control is supposed to achieve
- what the current environment appears to look like
- whether the control seems effective, partial, or in need of improvement
- what evidence was reviewed
- what should be done next to strengthen the control

I also included ownership, priority, status, and framework references so the assessment feels closer to how a real security or GRC team would track control health and remediation work.

This project helped me practice looking beyond control design and thinking more critically about control effectiveness, evidence, and follow-up actions.
