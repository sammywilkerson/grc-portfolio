This risk register was built as part of a GRC portfolio project based on a realistic SaaS company running in AWS and handling customer data.

I used this scenario because it reflects the kinds of risks many cloud-based organizations face every day: access issues, weak logging, vendor dependencies, and recovery concerns.

The goal of this artifact is to show how I think through risk by identifying issues, assigning ownership, prioritizing impact, and connecting each item to common control frameworks like NIST CSF, CIS Controls, and SOC 2.
